# 数据运营（上线后持续）

## 一、核心指标（KPI）

### 1.1 流量指标
| 指标 | 含义 | 健康值 |
|------|------|--------|
| 播放量 | 总播放次数 | 首周>100万 |
| 完播率 | 看完整集的比例 | >60% |
| 集间留存 | 看完第N集继续看N+1的比例 | >70% |
| 3秒留存 | 前3秒不划走的比例 | >50% |

### 1.2 付费指标
| 指标 | 含义 | 健康值 |
|------|------|--------|
| 付费率 | 看到付费点后付费的比例 | >3% |
| ARPU | 每付费用户平均消费 | >30元 |
| 付费转化集 | 哪一集付费率最高 | EP21（设计目标） |
| ROI | 投放回报率 | >1.5 |

### 1.3 互动指标
| 指标 | 含义 | 健康值 |
|------|------|--------|
| 点赞率 | 点赞/播放 | >5% |
| 评论率 | 评论/播放 | >1% |
| 分享率 | 分享/播放 | >2% |
| 收藏率 | 收藏/播放 | >3% |

## 三、关键数据分析场景

### 3.1 如果3秒留存低（<40%）
- **原因**：开头不够抓人
- **解法**：
  - 重新剪辑片头，前3秒直接放冲突
  - 加上文字钩子："她等了他23年，他来了——但她已经老了"
  - 测试不同封面图

### 3.2 如果完播率低（<50%）
- **原因**：中间拖沓/节奏慢
- **解法**：
  - 剪辑加速，删除多余空镜
  - 加背景音乐增强节奏感
  - 缩短单集时长

### 3.3 如果集间留存低（EP20→EP21 <50%）
- **原因**：付费点悬念不够强/定价过高
- **解法**：
  - 调整EP20结尾悬念
  - 降低首集付费价格
  - 提供"前3集付费内容免费试看"

### 3.4 如果付费率低（<2%）
- **原因**：免费集没有建立足够的情感粘性
- **解法**：
  - 检查EP1-20是否足够虐/爽
  - 在EP19-20增加更强的情感钩子
  - 调整付费点位置（可能提前到EP15或延后到EP25）

### 3.5 如果ROI<1（投放亏钱）
- **原因**：素材/定向/出价问题
- **解法**：
  - 更换投放素材（至少测试10条）
  - 调整定向人群
  - 降低出价，控制成本
  - 暂停投放，靠自然流量

## 五、迭代优化

### 5.1 基于数据的内容优化
- 找到完播率最高的3集 → 分析为什么好 → 复制到其他集
- 找到流失率最高的3集 → 分析问题 → 重新剪辑
- 找到评论最多的话题 → 制作相关切片 → 二次传播

### 5.2 A/B测试
- 同一集不同封面 → 测试点击率
- 同一片段不同文案 → 测试转化率
- 不同付费节点 → 测试付费率
