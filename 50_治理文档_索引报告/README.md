# docs 总入口

这里是 `AI_Short_Drama` 的文档治理入口。

## 建议阅读顺序

### 1. 先看项目全貌
- [PROJECT_STRUCTURE.md](PROJECT_STRUCTURE.md)

用途：
- 快速理解整个项目由哪些区域构成
- 区分正式主线、历史素材、过程文档、工具目录
- 了解大体量目录的风险点

### 2. 再看文档主入口
- [DOCUMENT_INDEX.md](DOCUMENT_INDEX.md)

用途：
- 找到正式主线文档
- 找到规范说明入口
- 找到历史素材区与过程文档区
- 明确哪些目录暂不建议直接清理

### 3. 先看最终总报告
- [FINAL_GOVERNANCE_REPORT.md](FINAL_GOVERNANCE_REPORT.md)

用途：
- 一页看完本轮整理做了什么
- 快速定位当前主线、风险区和下一步

### 4. 再看复核结论
- [DOCUMENT_REVIEW.md](DOCUMENT_REVIEW.md)
- [复核报告_一致性与重复候选.md](复核报告_一致性与重复候选.md)

用途：
- 查看主线剧名、人物、集数是否一致
- 查看命名疑点
- 查看重复/备份候选目录
